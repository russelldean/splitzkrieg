**DiscoverySearch.tsx** — `'use client'`

A two-mode search component for the home page ONLY (Header SearchBar stays unchanged per CONTEXT.md).

State machine with three states:
- IDLE: Input not focused, dropdown closed
- BROWSING: Input focused but query is empty — show category prompt cards
- SEARCHING: Input focused and query.length >= 2 — show Fuse.js fuzzy results

Category prompts (shown in BROWSING state):
```typescript
const categories = [
  { label: 'Bowlers', href: '/bowlers', description: 'Browse all bowlers', icon: /* person SVG */ },
  { label: 'Teams', href: '/teams', description: 'View team rosters', icon: /* people SVG */ },
  { label: 'Seasons', href: '/seasons', description: 'Explore 35+ seasons', icon: /* calendar SVG */ },
  { label: 'Leaderboards', href: '/leaderboards', description: 'All-time records', icon: /* trophy SVG */ },
];
```

Visual treatment:
- Larger than the header SearchBar — use `text-lg py-3 px-6` for the input
- Placeholder text: "Search bowlers by name..."
- Category prompts render as a dropdown panel below the input with icon, label, and description per row
- Each category is a Link from next/link — uses onMouseDown + e.preventDefault() pattern (same as SearchBar.tsx) to prevent blur race condition
- When typing, category prompts disappear and Fuse.js results appear (same result rendering as SearchBar.tsx but slightly larger)

Fuse.js integration:
- Load search index from `/api/search-index` on mount (same as SearchBar.tsx)
- Create Fuse instance with same config: `{ keys: ['name'], threshold: 0.3, minMatchCharLength: 2 }`
- Results: show bowler name, seasons active count, navigate to `/bowler/[slug]` on select
- Keyboard navigation: ArrowDown/ArrowUp/Enter/Escape (same as SearchBar.tsx)
- Full ARIA support: role="combobox", aria-expanded, aria-autocomplete, aria-controls, aria-activedescendant

Mobile: Full width, slightly smaller padding on mobile (`py-2 px-4` on sm:), category prompts stack vertically
