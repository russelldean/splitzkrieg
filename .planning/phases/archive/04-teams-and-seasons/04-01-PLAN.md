**Query Functions:**

1. `getAllTeamSlugs()` — SELECT slug FROM teams WHERE slug IS NOT NULL. Returns TeamSlug[].

2. `getTeamBySlug(slug: string)` — React.cache wrapped. SELECT teamID, teamName, slug FROM teams WHERE slug = @slug. Returns Team | null.

3. `getTeamCurrentRoster(teamID: number)` — Everyone who bowled for this team in the current season (use subquery for latest seasonID, same pattern as getCurrentSeasonID). Each bowler: name, slug, gamesBowled (COUNT(scoreID)*3), seasonAverage (SUM(scratchSeries)/NULLIF(COUNT(scoreID)*3, 0)). Filter isPenalty=0. Order by gamesBowled DESC. Returns TeamRosterMember[].

4. `getTeamSeasonByseason(teamID: number)` — One row per season this team has participated in. JOIN scores with seasons table. Group by seasonID. Include: season displayName, romanNumeral, compute seasonSlug as LOWER(REPLACE(displayName, ' ', '-')), team name at time from teamNameHistory (LEFT JOIN, fall back to current team name), total games (COUNT*3), total pins (SUM scratchSeries), team average, roster size (COUNT DISTINCT bowlerID), hasScheduleData (EXISTS check against schedule table for that seasonID). Order by season year DESC, period DESC (newest first). Filter isPenalty=0. Returns TeamSeasonRow[].

5. `getTeamSeasonBowlers(teamID: number, seasonID: number)` — Individual bowlers for one team+season. JOIN scores with bowlers. Group by bowlerID. Show games, pins, average. Order by games DESC. Filter isPenalty=0. Returns TeamSeasonBowler[].

6. `getTeamAllTimeRoster(teamID: number)` — Every bowler who ever bowled for this team across all seasons. Group by bowlerID. Show total games, total pins, average, seasons count (COUNT DISTINCT seasonID). Sorted by totalGames DESC (loyalty metric per user decision). Filter isPenalty=0. Returns AllTimeRosterMember[].

7. `getTeamFranchiseHistory(teamID: number)` — Query teamNameHistory table for this team. SELECT teamNameHistoryID, teamName, startDate, endDate. Order by teamNameHistoryID ASC (chronological). Returns FranchiseNameEntry[]. If table columns differ, adapt — the key fields are team name and optional date range.

8. `getAllTeamsDirectory()` — React.cache wrapped. All teams with aggregate stats. JOIN teams with scores to get rosterCount (COUNT DISTINCT bowlerID overall), seasonsActive (COUNT DISTINCT seasonID), totalGames, totalPins. Determine isActive by checking if team has scores in the current season (subquery). Order active first, then by teamName ASC. Filter isPenalty=0. Returns DirectoryTeam[].

**Important patterns to follow:**
- Use `getDb()` for every query (never cache the pool reference)
- Use `.input('paramName', value)` for parameterized queries (never string interpolation)
- Use `isPenalty = 0` filter on all scores-based queries
- Use `COUNT(sc.scoreID) * 3` for game counting (each scoreID = 3 games, one per night)
- Compute season slugs inline: `LOWER(REPLACE(s.displayName, ' ', '-')) AS seasonSlug`
- React.cache only for functions called from both generateMetadata and page component (getTeamBySlug, getAllTeamsDirectory)

**Query Functions:**

1. `getAllSeasonSlugs()` — SELECT LOWER(REPLACE(displayName, ' ', '-')) AS slug FROM seasons. Returns SeasonSlug[].

2. `getSeasonBySlug(slug: string)` — React.cache wrapped. Find season WHERE LOWER(REPLACE(displayName, ' ', '-')) = @slug. Return seasonID, displayName, romanNumeral, year, period, computed slug. Returns Season | null.

3. `getSeasonStandings(seasonID: number)` — Team standings for this season. JOIN scores with teams, LEFT JOIN seasonDivisions. Group by team. Compute: rosterSize, totalGames, totalPins, teamScratchAvg (SUM(scratchSeries)/NULLIF(COUNT*3, 0)), teamHcpAvg (SUM(handSeries)/NULLIF(COUNT*3, 0) — handSeries is a computed column in scores), weeksPlayed (MAX week). Order by divisionName, totalPins DESC. Filter isPenalty=0, teamID IS NOT NULL. Returns StandingsRow[].

4. `getSeasonLeaderboards(seasonID: number, gender: 'M' | 'F' | null, category: 'avg' | 'highGame' | 'highSeries' | 'totalPins' | 'games200' | 'series600' | 'turkeys')` — Top 10 bowlers for a given category. JOIN scores with bowlers and teams. Filter by gender if not null (WHERE b.gender = @gender). Compute the requested stat, rank by it, return TOP 10. Each category has different aggregation:
    - avg: SUM(scratchSeries) / NULLIF(COUNT(scoreID)*3, 0), minimum 9 games (3 nights)
    - highGame: MAX of game1, game2, game3 (use GREATEST or CASE-based MAX)
    - highSeries: MAX(scratchSeries)
    - totalPins: SUM(scratchSeries)
    - games200: COUNT games >= 200 (need to unpivot game1/2/3)
    - series600: COUNT WHERE scratchSeries >= 600
    - turkeys: SUM(turkeys)
    Filter isPenalty=0. Returns SeasonLeaderEntry[].

    **Implementation note:** Rather than one complex parameterized function, consider creating a helper `getSeasonLeaderboard(seasonID, gender, category)` that uses a switch/case to build the appropriate SQL. Or create separate focused queries and a wrapper. Use whichever approach keeps it readable — the user prefers readability (Phase 3 decision to use sequential queries over complex JOINs).

5. `getSeasonFullStats(seasonID: number)` — Every bowler who bowled in this season with full stats. JOIN scores with bowlers and teams. Group by bowler. Include gender for client-side filtering/splitting. Compute: gamesBowled, totalPins, scratchAvg, hcpAvg, highGame (MAX of game1/2/3), highSeries (MAX scratchSeries), games200Plus (unpivot and count), series600Plus, turkeys (SUM). Filter isPenalty=0. Order by scratchAvg DESC. Returns SeasonFullStatsRow[].

6. `getSeasonSchedule(seasonID: number)` — All schedule entries for this season. JOIN schedule with teams (home and away). Return week, matchDate, home/away team info with slugs. Order by week ASC, then by matchup. Returns SeasonScheduleWeek[].

7. `getSeasonRecords(seasonID: number)` — Individual records for the season (high scratch game, high scratch series, high hcp series, most turkeys, most 200 games). Each is a separate TOP 1 subquery or a series of queries. JOIN with bowlers for name/slug. Returns SeasonRecords.

8. `getAllSeasonsDirectory()` — React.cache wrapped. All seasons with summary stats. JOIN seasons with scores for bowlerCount (COUNT DISTINCT bowlerID) and teamCount (COUNT DISTINCT teamID). LEFT JOIN seasonChampions for champion name (will be null). Compute slug inline. Order by year DESC, period DESC (newest first). Filter isPenalty=0. Returns DirectorySeason[].

9. `getSeasonHeroStats(seasonID: number)` — Aggregate stats for the season hero: league average (all bowlers avg), total games, total bowlers, top average holder, high game holder, high series holder. Can be derived from getSeasonFullStats results in the page component, OR as a dedicated lightweight query. Prefer lightweight dedicated query for the hero to avoid loading full stats table data just for the hero.

**Important patterns:**
- Season slug is always computed inline: `LOWER(REPLACE(s.displayName, ' ', '-'))`
- For getSeasonBySlug, the WHERE clause compares computed slug to param — since there's no slug column, compute in WHERE: `WHERE LOWER(REPLACE(displayName, ' ', '-')) = @slug`
- Use handSeries (computed column in scores table) for handicap stats
- 200+ game counting requires unpivoting game1/game2/game3 — use a CTE or subquery with CROSS APPLY (VALUES (game1),(game2),(game3)) pattern
- Gender filtering: WHERE b.gender = @gender (naturally excludes NULL gender rows)
- React.cache only for getSeasonBySlug and getAllSeasonsDirectory
