# Phase 12 -- UI Design Contract

> Visual and interaction contract for the Navigation and Discoverability Overhaul. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, exit ramp icon gap |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, nudge component internal spacing |
| xl | 32px | Layout gaps between recap sections |
| 2xl | 48px | Major section breaks (recap to discovery section) |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets on exit ramps and nudges: minimum 44px height (D-23, 50+ year old bowlers on phones in dark bowling alley)
- TrailNav pills already use `px-3 py-1.5` (approx 36px height). New NextStopNudge must be larger: `px-4 py-3` minimum (approx 48px height)

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 16px (sm: 18px) | 400 | 1.625 | `text-base sm:text-lg leading-relaxed font-body` |
| Label / Exit Ramp | 14px | 600 | 1.4 | `text-sm font-semibold font-body` |
| Section Heading | 18px | 400 | 1.2 | `text-lg font-heading` |
| Page Heading | 24px | 400 | 1.2 | `text-2xl font-heading` |

Source: Existing blog-prose class, SectionHeading component, WeekRecap SectionHeader. Weight consolidation: project uses `font-semibold` (161 occurrences) over `font-medium` (67), so 400 + 600 is the canonical pair.

Phase-specific rules:
- NextStopNudge destination label: 16px font-heading (not body), to feel like a section title, not a utility link
- NextStopNudge "Keep exploring" label: 14px `text-sm uppercase tracking-wide text-navy/60 font-semibold`. The uppercase + tracking-wide treatment provides sufficient visual subordination without needing a smaller size.
- NextStopNudge description: 14px font-body at text-navy/70 (above 60% opacity floor)
- Exit ramp link text: 14px font-semibold, not font-heading (these are utility links, not content headings)
- RecapCallout (new feature announcement at top): 16px font-body font-semibold for the headline, 14px font-body for description

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAF7F2` (cream) | Page backgrounds, recap section backgrounds |
| Secondary (30%) | `#FFFFFF` (white) | Cards, ExploreLink blocks, NextStopNudge background |
| Foreground | `#1B2A4A` (navy) | All text, headings, icons |
| Accent (10%) | `#C53030` (red) | See reserved list below |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for:
- Exit ramp hover state (`hover:bg-red-600 hover:text-white` per existing SectionHeader pattern)
- SectionHeading underline bar (`bg-red-600/40`)
- RecapCallout "NEW" badge background (`bg-red-600 text-white`)
- Active/current step indicator in guided path
- Link hover color in blog prose (`text-red-600 hover:text-red-700`)

Accent NOT used for:
- NextStopNudge default state (uses navy/cream, accent only on hover)
- TrackVisibility wrappers (invisible, no visual treatment)
- Discovery section card borders at rest (navy/10, not red)

Opacity rules (per CLAUDE.md):
- Text minimum opacity: 60% (`text-navy/60` is the floor)
- Existing patterns to preserve: `text-navy/80` for section headings, `text-navy/65` for secondary descriptions
- New components must use `text-navy/70` or higher for all readable text

## Interaction Contracts

### ExitRamp

- **Default state:** `bg-navy/[0.06]` pill with navy/80 text, right chevron icon (3.5x3.5), rounded-lg
- **Hover state:** `bg-red-600 text-white` with chevron translate-x 2px (existing SectionHeader pattern)
- **Touch target:** minimum 44px total height (py-2 px-3 on mobile, can be py-1.5 px-3 on desktop)
- **Click behavior:** fires `posthog.capture('exit_ramp_clicked', { section, destination, link_text, source_page })` then navigates
- **Placement:** Immediately below each condensed recap section heading, inline with content flow

### NextStopNudge

- **Layout:** Full-width card with white background, `border border-navy/10`, `rounded-xl`, `p-4 sm:p-5`
- **Content:** Left side: "Keep exploring" label (14px uppercase tracking-wide font-semibold text-navy/60) + destination title (16px font-heading text-navy) + description (14px font-body text-navy/70). Right side: right arrow icon (20x20).
- **Hover state:** `shadow-md border-red-600/30` (matches existing ExploreLink hover pattern)
- **Touch target:** entire card is tappable, minimum 64px height
- **Placement:** After page content, before TrailNav. Separated by `mt-8` from content.
- **Sequential path (D-19):** week results -> season standings -> season stats/leaderboards -> milestones -> all-time records
- **Contextual:** shows different destination based on current page. If no "next" in sequence, shows homepage.

### RecapCallout

- **Layout:** `bg-red-600/[0.08] border-l-4 border-red-600 rounded-r-lg p-4`
- **Content:** "NEW" badge (inline, `bg-red-600 text-white text-xs font-bold px-2 py-0.5 rounded-full`) + headline (16px font-semibold) + description (14px text-navy/75) + link
- **Placement:** First element in recap, before awards section
- **Condition:** Only renders when `callout` prop is provided (not every week has a new feature)

### DiscoverySection

- **Layout:** Replaces current Keep Exploring grid. 2-column grid on desktop, single column on mobile.
- **Stable links (always present):** "Find Your Profile" (search), "All-Time Records" (/stats/all-time)
- **Rotating highlights:** 2 items pulled from latest entries in content/updates.ts. Rotate per-build, not per-visit (avoid hydration mismatch).
- **Card style:** Matches existing ExploreLink pattern exactly: `bg-white border border-navy/10 rounded-lg p-3 shadow-sm hover:shadow-md hover:border-red-600/30`

### TrackVisibility Wrapper

- **Visual:** Completely invisible. No border, background, padding, or layout shift.
- **Behavior:** Wraps a server-rendered section in a client `<div ref={ref}>`. Fires `section_viewed` event when 30% visible. Fires once per page load, then disconnects observer.
- **Sections to wrap on bowler profile:** YouAreAStar, GameProfile, PersonalRecordsPanel, MilestoneWatch, AverageProgressionChart
- **Sections to wrap on recap:** each condensed section (results, standings, leaderboards, milestones), DiscoverySection

### Bowler Profile Reorder (D-07)

New section order:
1. BowlerHero (unchanged)
2. LastWeekHighlight (unchanged)
3. PersonalRecordsPanel (unchanged)
4. MilestoneWatch (unchanged)
5. **YouAreAStar** (moved up from position 8)
6. **GameProfile** (moved up from position 9)
7. AverageProgressionChart (was position 5)
8. SeasonStatsTable (was position 6)
9. GameLog (was position 7)

Rationale: YouAreAStar and GameProfile are personality content with universal appeal (D-07). The "Hogwarts house" analogy from CONTEXT.md confirms these should be early in the page, not buried below stat tables that only deep diggers reach.

## Mobile / Bowling Alley Readability (D-23, D-24, D-25)

This phase does NOT implement a separate dark mode or bowling alley mode. Per D-24, this is a "consider" item. The contract ensures all new components meet readability requirements in a dark bowling alley on a phone:

| Requirement | Implementation |
|-------------|---------------|
| High contrast | All text at navy/60 minimum opacity on cream background. White card backgrounds provide strong contrast. |
| Large tap targets | ExitRamp: 44px min height. NextStopNudge: 64px min height (entire card tappable). |
| No tiny text | Smallest text in new components: 14px (labels, descriptions). All content text 14px+. |
| Readable on small screens | Single column layout on mobile for all new components. No horizontal scrolling. |

## PostHog Event Schema

| Event Name | Properties | Fired By |
|------------|------------|----------|
| `exit_ramp_clicked` | `section`, `destination`, `link_text`, `source_page` | ExitRamp component |
| `section_viewed` | `section`, `page` | TrackVisibility wrapper |
| `next_stop_clicked` | `current_page`, `destination`, `nudge_label` | NextStopNudge component |
| `discovery_link_clicked` | `link_title`, `link_type` (stable/rotating), `source_page` | DiscoverySection links |

Event naming follows PostHog convention: `[noun]_[past_tense_verb]`.
