**ExitRamp.tsx** (per D-12, UI-SPEC interaction contract):
- 'use client' component
- Props: `{ href: string; section: string; linkText: string; className?: string }`
- Uses `usePostHog` from 'posthog-js/react' to capture `exit_ramp_clicked` with properties: `{ section, destination: href, link_text: linkText, source_page: window.location.pathname }`
- Renders a Next.js `<Link>` with the exact styling from the existing SectionHeader link pattern in WeekRecap.tsx:
  `inline-flex items-center gap-1.5 px-3 py-1.5 rounded-lg bg-navy/[0.06] font-body text-sm font-semibold text-navy/80 hover:bg-red-600 hover:text-white transition-colors`
  (Note: font-semibold per UI-SPEC, not font-medium from existing SectionHeader)
- Touch target: add `min-h-[44px] flex items-center` on mobile for D-23 compliance
- Includes right chevron SVG icon (w-3.5 h-3.5), same as existing SectionHeader
- The onClick handler calls posthog.capture() then lets the Link navigate normally (no e.preventDefault)

**TrackVisibility.tsx** (per D-13, UI-SPEC interaction contract):
- 'use client' component
- Props: `{ section: string; page: string; children: React.ReactNode; className?: string }`
- Uses IntersectionObserver with `{ threshold: 0.3 }` to detect when 30% visible
- Fires `posthog.capture('section_viewed', { section, page })` exactly once per page load
- After firing, disconnects the observer (fire-once pattern)
- Uses `useRef<boolean>(false)` for the tracked flag
- Renders `<div ref={ref} className={className}>{children}</div>` with no visual treatment (invisible wrapper)
- Cleanup: return observer.disconnect() from useEffect

**useTrackVisibility.ts** (hook version for components that need the ref directly):
- 'use client' at top
- Signature: `function useTrackVisibility(sectionName: string, page: string): React.RefObject<HTMLDivElement | null>`
- Same IntersectionObserver logic as TrackVisibility but returns the ref
- Uses `usePostHog` hook for posthog instance
- Fire-once with tracked useRef boolean
- Cleanup on unmount
