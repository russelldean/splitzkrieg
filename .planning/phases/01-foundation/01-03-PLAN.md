  <action>
  **Install Fuse.js:**
  ```bash

  ```

  **src/lib/search-index.ts** — Build-time search index generation:

  Define the `SearchEntry` interface:
  ```typescript
  export interface SearchEntry {
    id: number;
    name: string;
    slug: string;
    seasonsActive: number;
  }
  ```

  Create `generateSearchIndex()` function:
  - Import `getDb` from `./db`.
  - Query Azure SQL for all bowlers: SELECT bowlerID, firstName, lastName, and count of distinct seasons from roster entries.
  - Generate slug from firstName + lastName (lowercase, spaces to hyphens). Use the SAME slug generation logic as queries.ts getAllBowlerSlugs to ensure consistency.
  - Return array of SearchEntry objects.
  - This function runs at build time only (called from a static Route Handler).

  **Then create a static Route Handler** at `src/app/api/search-index/route.ts`:
  - Export async `GET` function.
  - Call `generateSearchIndex()`.
  - Return `NextResponse.json(entries)`.
  - Add `export const dynamic = 'force-static'` so Next.js pre-renders this at build time — the JSON is served as a static file, no server function needed at runtime.

  This approach avoids needing a pre-build script. The route handler runs during `next build` and the output is cached as static JSON.

<task type="auto">
  <name>Task 2: Wire Fuse.js into SearchBar with autocomplete dropdown</name>
  <files>src/components/layout/SearchBar.tsx</files>
  <action>
  **Upgrade SearchBar.tsx** from visual placeholder to working fuzzy search:

  This is a 'use client' component. Replace the placeholder implementation entirely.

  - Import `useState`, `useEffect`, `useMemo`, `useRef` from React.
  - Import `Fuse` from `fuse.js`.
  - Import `useRouter` from `next/navigation`.
  - Import `SearchEntry` type from `@/lib/search-index`.

  **State:**
  - `query`: string (search input value)
  - `entries`: SearchEntry[] (loaded from API)
  - `isOpen`: boolean (dropdown visibility)
  - `selectedIndex`: number (keyboard navigation, -1 = none)

  **On mount:** Fetch `/api/search-index` and set entries. Use useEffect with empty deps.

  **Fuse instance:** Create with `useMemo` — keys: ['name'], threshold: 0.3, minMatchCharLength: 2. Recreate when entries change.

  **Results:** When query.length > 1, call `fuse.search(query, { limit: 8 })`. Otherwise empty array.

  **Keyboard navigation:**
  - ArrowDown: increment selectedIndex (wrap to 0)
  - ArrowUp: decrement selectedIndex (wrap to last)
  - Enter: navigate to selected result's bowler page
  - Escape: close dropdown, clear query

  **Dropdown UI:**
  - Absolute positioned below the input (`top-full mt-1`).
  - White background, rounded-lg, shadow-lg, border border-navy/10, z-50.
  - Each result: block px-4 py-2 hover:bg-cream-dark font-body text-navy.
  - Selected item (via keyboard): bg-cream-dark to show active state.
  - Show seasons active count in muted text: `<span className="text-navy/40 text-xs">{item.seasonsActive} seasons</span>`.

  **Click outside:** Use useRef on the container + useEffect with document click listener to close dropdown when clicking outside.

  **Navigation:** On result click or Enter on selected, use `router.push(/bowler/${item.slug})`. Close dropdown and clear query after navigation.

  **Accessibility:**
  - Input: role="combobox", aria-expanded, aria-autocomplete="list", aria-controls pointing to results list id.
  - Results list: role="listbox" with unique id.
  - Each result: role="option", aria-selected for active item.
  </action>
