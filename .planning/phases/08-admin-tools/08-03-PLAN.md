1. Create `src/lib/admin/lineups.ts`:
   - `getAllBowlers(): Promise<Array<{bowlerID: number, firstName: string, lastName: string}>>` -- query all bowlers for captain picker
   - `getRecentRoster(teamID: number, seasonID: number): Promise<number[]>` -- bowlerIDs from most recent scores for this team, used to pre-sort the bowler picker
   - `submitLineup(seasonID: number, week: number, teamID: number, submittedBy: string, entries: Array<{position: number, bowlerID: number | null, newBowlerName: string | null}>): Promise<number>` -- INSERT into lineupSubmissions + lineupEntries, return submission ID. If a submission already exists for this team/season/week, replace it.
   - `getLineups(seasonID: number, week: number): Promise<LineupSubmission[]>` -- all submissions for a week with entries and team names joined
   - `editLineup(submissionID: number, entries: Array<{position: number, bowlerID: number | null, newBowlerName: string | null}>): Promise<void>` -- admin edits, update status to 'edited'
   - `pushLineupsToLP(cookie: string, seasonID: number, week: number): Promise<{pushed: number, errors: string[]}>` -- push finalized lineups to LP API. Refactor from scripts/update-leaguepals.mjs. Update status to 'pushed'.
   - `getLastWeekLineup(teamID: number, seasonID: number, currentWeek: number): Promise<LineupEntry[]>` -- get previous week's lineup for default pre-fill

2. Create `src/app/api/admin/magic-link/route.ts` POST:
   - Accepts `{ teamID: number, captainName: string, captainEmail: string }` from admin
   - Calls `signToken({ role: 'captain', teamID, captainName }, '90d')` for long-lived session
   - Stores in captainSessions table (for revocation tracking)
   - Sends magic link email via Resend: `https://splitzkrieg.com/lineup/login?token={token}`
   - Returns 200 with `{ sent: true }`
   - Requires admin auth

3. Create `src/app/api/lineup/auth/route.ts` GET:
   - Reads `token` query param
   - Verifies JWT, checks captainSessions table for revocation
   - Sets `lineup-token` httpOnly cookie (path=/, sameSite=lax, secure in prod, maxAge=90 days)
   - Redirects to `/lineup`

4. Create `src/app/api/lineup/submit/route.ts` POST:
   - Accepts `{ seasonID: number, week: number, entries: Array<{position, bowlerID, newBowlerName}> }`
   - Reads captain identity from `lineup-token` cookie (teamID, captainName)
   - Calls `submitLineup()`
   - Returns 200 with submission ID

5. Create `src/app/api/admin/lineups/route.ts`:
   - GET: returns all lineups for given seasonID/week query params
   - PUT: accepts `{ submissionID: number, entries: [...] }`, calls `editLineup()`
   - Requires admin auth

6. Create `src/app/api/admin/lineups/push/route.ts` POST:
   - Accepts `{ cookie: string, seasonID: number, week: number }`
   - Calls `pushLineupsToLP()`
   - Returns push results
   - Requires admin auth
