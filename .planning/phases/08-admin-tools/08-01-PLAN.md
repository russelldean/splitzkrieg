  <action>
1. Create `scripts/create-admin-tables.mjs` -- a migration script that creates 4 new tables in Azure SQL. Follow the env-loading pattern from existing scripts (read `.env.local`, parse lines). Tables:
   - `blogPosts` (id INT IDENTITY PK, slug VARCHAR(255) UNIQUE NOT NULL, title NVARCHAR(500) NOT NULL, content NVARCHAR(MAX) NOT NULL, excerpt NVARCHAR(500), type VARCHAR(50) DEFAULT 'recap', seasonRomanNumeral VARCHAR(20), seasonSlug VARCHAR(100), week INT, heroImage VARCHAR(500), heroFocalY FLOAT, publishedAt DATETIME2 nullable, createdAt DATETIME2 DEFAULT GETDATE(), updatedAt DATETIME2 DEFAULT GETDATE())
   - `lineupSubmissions` (id INT IDENTITY PK, seasonID INT NOT NULL, week INT NOT NULL, teamID INT NOT NULL FK to teams, submittedBy VARCHAR(100), submittedAt DATETIME2 DEFAULT GETDATE(), status VARCHAR(20) DEFAULT 'submitted')
   - `lineupEntries` (id INT IDENTITY PK, submissionID INT NOT NULL FK to lineupSubmissions, position INT NOT NULL, bowlerID INT nullable, newBowlerName VARCHAR(200) nullable)
   - `captainSessions` (id INT IDENTITY PK, teamID INT NOT NULL FK to teams, captainName VARCHAR(100) NOT NULL, captainEmail VARCHAR(255) NOT NULL, token VARCHAR(500) NOT NULL, createdAt DATETIME2 DEFAULT GETDATE(), expiresAt DATETIME2 NOT NULL, revoked BIT DEFAULT 0)
   Script should use IF NOT EXISTS checks so it's idempotent. Run it after creation.

2. Create `src/lib/admin/types.ts` -- shared TypeScript types for the entire admin system:
   - `TokenPayload { role: 'admin' | 'captain'; teamID?: number; captainName?: string; }`
   - `StagedBowler { bowlerID: number | null; bowlerName: string; teamID: number; teamName: string; game1: number | null; game2: number | null; game3: number | null; turkeys: number; incomingAvg: number | null; isPenalty: boolean; isUnmatched: boolean; matchedSuggestions?: Array<{bowlerID: number; name: string; score: number}>; }`
   - `StagedMatch { matchID?: number; homeTeamID: number; homeTeamName: string; awayTeamID: number; awayTeamName: string; bowlers: StagedBowler[]; }`
   - `ValidationWarning { bowlerID: number | null; bowlerName: string; field: string; message: string; severity: 'info' | 'warning'; }`
   - `PipelineStatus { step: 'idle' | 'pulled' | 'reviewing' | 'confirmed' | 'published'; seasonID: number; week: number; matchCount: number; }`
   - `BlogPost { id: number; slug: string; title: string; content: string; excerpt: string | null; type: 'recap' | 'announcement'; seasonRomanNumeral: string | null; seasonSlug: string | null; week: number | null; heroImage: string | null; heroFocalY: number | null; publishedAt: string | null; createdAt: string; updatedAt: string; }`
   - `LineupSubmission { id: number; seasonID: number; week: number; teamID: number; teamName?: string; submittedBy: string | null; submittedAt: string; status: 'submitted' | 'edited' | 'pushed'; entries: LineupEntry[]; }`
   - `LineupEntry { id: number; submissionID: number; position: number; bowlerID: number | null; newBowlerName: string | null; bowlerName?: string; }`

3. Create `src/lib/admin/auth.ts` -- JWT auth helpers using `jose` library:
   - `signToken(payload: TokenPayload, expiresIn?: string): Promise<string>` -- defaults to '30d', uses HS256 with ADMIN_JWT_SECRET env var
   - `verifyToken(token: string): Promise<TokenPayload | null>` -- returns null on any error (expired, invalid, etc.)
   - `requireAdmin(request: NextRequest): Promise<TokenPayload>` -- reads 'admin-token' cookie, verifies, throws if not admin role. Used in API routes.
   - `requireCaptain(request: NextRequest): Promise<TokenPayload>` -- reads 'lineup-token' cookie, verifies, throws if not captain role.
   - Use `new TextEncoder().encode(process.env.ADMIN_JWT_SECRET!)` for the secret key.

4. Create `proxy.ts` at project root (Next.js 16 replaces middleware.ts):
   - Match paths: `/admin/:path*`, `/lineup/:path*`, `/api/admin/:path*`
   - For `/admin/*` (except `/admin/login`): check `admin-token` cookie exists, redirect to `/admin/login` if missing
   - For `/lineup/*` (except `/lineup/login`): check `lineup-token` cookie exists, redirect to `/lineup/login?expired=1` if missing
   - For `/api/admin/*`: check `admin-token` cookie exists, return 401 JSON if missing
   - Export both the `proxy` function and `config` with matcher array
   - Note: proxy.ts does cookie existence checks only (lightweight). Full JWT verification happens in layout/API routes.

Install `jose` package: `npm install jose`

  <action>
1. Create `src/app/api/admin/auth/login/route.ts`:
   - POST endpoint accepting `{ password: string }` JSON body
   - Compare against `process.env.ADMIN_PASSWORD` (simple string comparison -- single admin user)
   - On match: call `signToken({ role: 'admin' })`, set `admin-token` httpOnly cookie (path=/admin, sameSite=lax, secure in production, maxAge=30 days), return 200
   - On mismatch: return 401 `{ error: 'Invalid password' }`

2. Create `src/app/api/admin/auth/logout/route.ts`:
   - POST endpoint that clears the `admin-token` cookie (set maxAge=0), returns 200

3. Create `src/app/admin/login/page.tsx`:
   - Client component ('use client')
   - Simple password form: single password input + "Log In" button
   - Style with Tailwind using the project's cream/navy/red palette
   - On submit: POST to `/api/admin/auth/login`, on success redirect to `/admin` using `router.push`
   - Show error message on 401
   - This page does NOT use the admin layout (no sidebar)
   - Add a "Splitzkrieg Admin" heading

4. Create `src/app/admin/layout.tsx`:
   - Server component that wraps all admin pages (except login which has its own layout)
   - Verify JWT on every load: read `admin-token` cookie via `cookies()`, call `verifyToken()`, redirect to `/admin/login` if invalid
   - Left sidebar nav with links: Dashboard (/admin), Scores (/admin/scores), Blog (/admin/blog), Lineups (/admin/lineups), Scoresheets (/admin/scoresheets)
   - Logout button in sidebar that POSTs to `/api/admin/auth/logout` and redirects
   - Navy sidebar, cream content area, responsive (sidebar collapses on mobile to top nav or hamburger)
   - Use the project's DM Serif Display for headings, Inter for body
