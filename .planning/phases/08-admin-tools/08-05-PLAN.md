  <action>
Install: `npm install jspdf jspdf-autotable`

1. Create `src/lib/admin/scoresheets.ts`:
   - `generateScoresheet(matches: ScoresheetMatch[]): jsPDF` -- per research code example
   - `ScoresheetMatch { homeTeamName: string; awayTeamName: string; week: number; date: string; bowlers: Array<{name: string; side: 'home' | 'away'; incomingAvg: number | null; handicap: number | null}> }`
   - One page per match (landscape letter)
   - Header: "Team A vs Team B", "Week N | Date"
   - Two tables per page (home team, away team)
   - Columns: #, Bowler, Avg, HCP, Game 1, Game 2, Game 3, Turkeys (last 4 blank for hand-writing)
   - HCP calculated using project formula: `FLOOR((225 - FLOOR(incomingAvg)) * 0.95)`. New bowlers (no avg) show 'NEW' for avg and '-' for HCP.
   - `getMatchupsForWeek(seasonID: number, week: number): Promise<ScoresheetMatch[]>` -- query schedule for matchups, join with lineup submissions or last week's scores to get bowler names + averages

2. Create `src/app/api/admin/scoresheets/route.ts` POST:
   - Accepts `{ seasonID: number, week: number }`
   - Calls `getMatchupsForWeek()` to get matchup data with bowler info
   - Calls `generateScoresheet()` to create PDF
   - Returns PDF as binary response (Content-Type: application/pdf, Content-Disposition: attachment)
   - Requires admin auth

3. Create `src/app/api/admin/publish/route.ts` POST:
   - Accepts `{ seasonID: number, week: number }`
   - Updates leagueSettings: SET publishedSeasonID and publishedWeek (same logic as scripts/publish-week.mjs)
   - Bumps `.data-versions.json` (scores channel)
   - Writes `.published-week` file
   - Calls POST /api/revalidate with REVALIDATION_SECRET to trigger ISR revalidation
   - Returns `{ published: true, week }`
   - Requires admin auth
   - The publish step is SEPARATE from score confirmation (per CONTEXT.md locked decision). Blog post happens between confirm and publish.

4. Create `src/app/api/admin/email/route.ts` POST:
   - Accepts `{ seasonID: number, week: number }`
   - Refactor logic from scripts/send-recap-email.mjs
   - Query subscriber list (or use hardcoded list from existing script)
   - Build inline HTML email with navy/cream/red palette (same as existing script)
   - Include weekly highlights, scores summary, link to blog post
   - Send via Resend. Use `russ@splitzkrieg.com` or `commissioner@splitzkrieg.com` as from address (Resend domain already verified). Set Reply-To to commissioner's personal email if configured in env.
   - Returns `{ sent: number, errors: string[] }`
   - Requires admin auth
