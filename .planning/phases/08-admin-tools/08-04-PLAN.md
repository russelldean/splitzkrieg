1. Create `src/lib/admin/blog-db.ts` -- blog CRUD against Azure SQL:
   - `getAllBlogPosts(): Promise<BlogPost[]>` -- all posts, newest first by createdAt
   - `getPublishedBlogPosts(): Promise<BlogPost[]>` -- where publishedAt IS NOT NULL, ordered by publishedAt DESC
   - `getBlogPostBySlug(slug: string): Promise<BlogPost | null>` -- single post by slug (for public pages)
   - `getBlogPostById(id: number): Promise<BlogPost | null>` -- single post by ID (for admin editor)
   - `createBlogPost(data: Omit<BlogPost, 'id' | 'createdAt' | 'updatedAt'>): Promise<number>` -- INSERT, return new ID
   - `updateBlogPost(id: number, data: Partial<BlogPost>): Promise<void>` -- UPDATE specific fields, always set updatedAt = GETDATE()
   - `deleteBlogPost(id: number): Promise<void>` -- DELETE by ID
   - Use `getDb()` and direct mssql queries with parameterized inputs (project pattern)
   - Use `withRetry()` for all DB calls

2. Create API routes:
   - `src/app/api/admin/blog/route.ts`: GET (list all posts for admin), POST (create new post, accepts BlogPost fields). Requires admin auth.
   - `src/app/api/admin/blog/[id]/route.ts`: GET (single post by ID), PUT (update post), DELETE (delete post). Requires admin auth.
   - `src/app/api/admin/blog/auto-draft/route.ts` POST: accepts `{ seasonID: number, week: number }`, queries confirmed scores for that week, generates a draft markdown body with stat block tables pre-populated (reuse the existing weekly recap stat components/format from blog post templates). Narrative text sections left as placeholder comments (e.g., "<!-- Write your recap here -->"). Returns `{ title, slug, content, type, seasonRomanNumeral, seasonSlug, week }`.

3. Create `scripts/migrate-blog-to-db.mjs`:
   - Read all MDX files from `content/blog/`
   - Parse frontmatter with gray-matter
   - INSERT each into blogPosts table (slug, title, content = raw MDX body, excerpt, type, seasonRomanNumeral, seasonSlug, week, heroImage, heroFocalY, publishedAt = frontmatter date)
   - Run idempotently (skip if slug already exists in DB)
   - Follow env-loading pattern from existing scripts
   - Run the script after creation.

4. Update `src/lib/blog.ts`:
   - Change `getAllPosts()` to read from DB via `getPublishedBlogPosts()` instead of filesystem
   - Change `getPostBySlug()` to read from DB via `getBlogPostBySlug()` instead of filesystem
   - Keep the PostMeta interface for backwards compatibility (map BlogPost -> PostMeta)
   - The PostMeta.date field maps from BlogPost.publishedAt
   - Content is now raw markdown (not MDX with imports), render with next-mdx-remote or react-markdown

5. Update `src/app/blog/page.tsx` and `src/app/blog/[slug]/page.tsx`:
   - These should now work with the updated blog.ts (DB-backed)
   - Ensure `generateStaticParams` queries blog slugs from DB (for static generation)
   - Content rendering: use MDXRemote (already in project) for backward compatibility with existing MDX content, but new posts will be plain markdown
   - Verify existing blog URLs still work after migration
