Add a new query function for weekly match scores:

```typescript
export interface WeeklyMatchScore {
  week: number;
  matchDate: string | null;
  teamID: number;
  teamName: string;
  teamSlug: string;
  bowlerID: number;
  bowlerName: string;
  bowlerSlug: string;
  game1: number | null;
  game2: number | null;
  game3: number | null;
  scratchSeries: number | null;
  handSeries: number | null;
  incomingAvg: number | null;
  incomingHcp: number | null;
}
```

`getSeasonWeeklyScores(seasonID: number)` — All individual bowler scores for all weeks in this season. JOIN scores with bowlers, teams, and LEFT JOIN schedule for matchDate. Include game1/2/3, scratchSeries, handSeries, incomingAvg, incomingHcp. Filter isPenalty=0. Order by week ASC, teamID, bowlerName. Returns WeeklyMatchScore[].

**Create `src/components/season/StandingsRaceChart.tsx`** (client component, 'use client'):
- F1-style position chart showing team rankings over the weeks.
- Props: weeklyScores (WeeklyMatchScore[]), standings (StandingsRow[]).
- **Data transformation:**
  1. For each week, compute cumulative team total pins up to that week.
  2. Rank teams by cumulative pins at each week (rank 1 = most pins).
  3. Transform into Recharts data: `{ week: 1, "Team A": 1, "Team B": 3, ... }` where value is the rank position.
- **Chart:**
  - Recharts LineChart with ResponsiveContainer.
  - X axis: week numbers.
  - Y axis: rank position, **reversed** (rank 1 at top, max rank at bottom). Use `reversed={true}` on YAxis domain.
  - One Line per team, each with a distinct color. Use team colors if possible, or cycle through a palette (navy shades, reds, greens, etc.).
  - Tooltip showing team name and rank for hovered week.
  - Legend showing team names.
  - If more than 10 teams, consider only showing top 8 or making legend scrollable to avoid visual clutter.
- Only render for seasons with schedule data (XXVI-XXXV) and when there are at least 3 weeks of data.
- If too complex to look good (too many overlapping lines), the user said to fall back to a simpler approach. Start with the line chart — if it works for the number of teams, ship it.

**Create `src/components/team/TeamTimeline.tsx`** (client component or server component):
- Grid visualization showing which teams existed in which seasons.
- Props: timelineData — pre-computed from page component.
- **Data needed:** For each team, which seasons they had scores in. The page component can derive this from directory data + a new lightweight query, OR compute from existing data.
- Actually, this needs a new query OR we can pass it from existing data. Let's add a lightweight query:

  Add to queries.ts: `getTeamSeasonPresence()` — Returns array of `{ teamID: number, teamName: string, slug: string, seasonID: number, seasonSlug: string, romanNumeral: string }`. Query: SELECT DISTINCT t.teamID, t.teamName, t.slug, s.seasonID, LOWER(REPLACE(s.displayName,' ','-')) AS seasonSlug, s.romanNumeral FROM scores sc JOIN teams t ON sc.teamID=t.teamID JOIN seasons s ON sc.seasonID=s.seasonID WHERE sc.isPenalty=0 AND sc.teamID IS NOT NULL ORDER BY t.teamName, s.year, s.period.

- **UI approach:** Styled HTML grid/table per research recommendation (NOT Recharts — a presence/absence grid is naturally a table).
  - Rows: teams (sorted alphabetically or by first appearance).
  - Columns: seasons (chronological, newest at left or right — user prefers reverse chronological, so newest at left).
  - Cells: Colored if team was active that season, empty if not.
  - Active cell color: navy bg with slight opacity. Hover to show team name + season name.
  - Current season cells: highlight with red accent.
  - Team names in first column link to /team/[slug].
  - Season headers link to /season/[slug].
- Horizontal scroll for the full timeline (35 seasons won't fit on mobile).
- Section heading: "League Timeline" or "Team History".
- Place at bottom of /teams page per user decision ("at bottom of page showing which teams existed in which seasons").
