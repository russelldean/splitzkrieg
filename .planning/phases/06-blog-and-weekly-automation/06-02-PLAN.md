   a. `getTopPerformers(seasonID: number, week: number)` — Top 5 scratch series, top 5 scratch game (single game max of game1/game2/game3), top 5 hcp series (handSeries). Include bowlerName, slug, gender. Use scores table joined with bowlers. Filter `isPenalty = 0`.

   b. `getWeekMilestones(seasonID: number, week: number)` — Career-high scratch series (compare this week's scratchSeries to max scratchSeries from all prior weeks across ALL seasons for that bowler). Career-high scratch game (compare max of hcpGame1-hcpGame3 minus... wait — scratch games are game1, game2, game3; hcpGame is handicap game). Actually: scratch games are `game1`, `game2`, `game3` columns. Check db-schema. Also detect first 200+ game, first 600+ series for a bowler (no prior week had one). Include bowlerName, slug.

   c. `getMatchResultsSummary(seasonID: number, week: number)` — From matchResults table: team wins/losses/ties, total team series (scratch and handicap). Identify sweeps (team won all 3 points or total). Include team names with links. Join with `teams` for team slug.

   d. `getStandingsSnapshot(seasonID: number, week: number)` — Current standings using teamPoints table (or compute from matchResults). Show rank, team name, total points, and movement vs previous week (up/down/same arrows). Join with `teams` for team slug.

   IMPORTANT: Include `JSON.stringify({seasonID, week})` in the `sql` option for cachedQuery so different season/week combos get different cache keys. Use pattern: `cachedQuery('queryName', fn, null, { sql: SQL_STRING + JSON.stringify({seasonID, week}) })`.

2. Create four async server components in `src/components/blog/`. Each receives `season` (roman numeral string) and `week` (number) as props. Each:
   - Resolves seasonID from the roman numeral (query or lookup)
   - Calls its query function from `src/lib/queries/blog.ts`
   - Renders a styled stat block card (white bg, rounded, border, shadow-sm — matching site design)
   - Uses score color coding from `src/lib/score-utils.ts` for individual scores
   - Links all bowler names to `/bowler/{slug}` and team names to `/team/{slug}`
   - Has a clear heading (e.g., "Top Performers", "Milestones & Personal Bests", "Match Results", "Standings")

   Specific component details:
   - **TopPerformers**: Three ranked lists (scratch series, scratch game, hcp series) in a grid. Show rank number, bowler name (linked), score value (color coded).
   - **MilestonesBlock**: List of milestone achievements. Each row: bowler name (linked), achievement text (e.g., "New career-high series: 645"). If no milestones this week, show "No new milestones this week" gracefully.
   - **MatchResultsSummary**: Table or grid of matchups. Each row: Team A vs Team B, series scores, winner indicator. Highlight sweeps. Link team names.
   - **StandingsSnapshot**: Standings table with rank, team name (linked), points, and movement arrow (green up arrow, red down arrow, gray dash for no change). Compare to previous week's standings to compute movement.
