1. Create `scripts/create-league-settings.mjs` — one-time setup script:
   ```sql
   CREATE TABLE leagueSettings (
     settingKey VARCHAR(50) PRIMARY KEY,
     settingValue VARCHAR(255) NOT NULL
   );
   INSERT INTO leagueSettings (settingKey, settingValue) VALUES ('publishedWeek', '4');
   INSERT INTO leagueSettings (settingKey, settingValue) VALUES ('publishedSeasonID', '35');
   ```
   Use the same DB connection pattern as `import-week-scores.mjs`. Include IF NOT EXISTS guard on the CREATE TABLE. Run this script once to create the table.

2. Create `scripts/publish-week.mjs`:
   - Usage: `node scripts/publish-week.mjs --week=5` or `node scripts/publish-week.mjs --week=5 --season=35`
   - Parse args from process.argv (same pattern as existing scripts)
   - Default season to current (35) if not specified
   - UPDATE leagueSettings SET settingValue = @val WHERE settingKey = 'publishedWeek'
   - UPDATE leagueSettings SET settingValue = @val WHERE settingKey = 'publishedSeasonID'
   - Print confirmation: "Published: Season {seasonID}, Week {week}"
   - Also print reminder: "Remember to bust cache and redeploy for changes to take effect"

3. Modify `src/lib/queries/home.ts` — replace `MAX(sc.week)` with publish gate in SNAPSHOT queries:
   - The `SNAPSHOT_STATS_SQL` and related snapshot queries use `WITH latestWeek AS (SELECT MAX(sc.week) AS wk FROM scores sc WHERE sc.seasonID = @seasonID AND sc.isPenalty = 0)`
   - Replace with: `WITH latestWeek AS (SELECT CAST(settingValue AS INT) AS wk FROM leagueSettings WHERE settingKey = 'publishedWeek')`
   - This changes the SQL text, which auto-invalidates the cache hash (per CLAUDE.md — no manual cache busting needed)
   - CRITICAL: Do NOT modify queries used by `generateStaticParams` — those iterate all seasons. Only modify the snapshot/homepage queries that show "latest" data.
   - The `getWeeklyHighlights` function also uses MAX(week) — update that too since it powers the homepage ticker.

4. Modify `src/lib/queries/bowlers.ts` — gate profile stat queries:
   - Find queries that use `MAX(sc2.week)` to determine the latest week for bowler stats display
   - Replace with the publishedWeek lookup from leagueSettings
   - CRITICAL: Only modify queries that determine what the LATEST week is for display. Do NOT modify historical queries that aggregate across all weeks.
   - Same cache invalidation applies — SQL text change = new hash

NOTE on cache: Per CLAUDE.md, changing SQL text auto-invalidates via hash. No `/* vN */` comments needed. No cache file deletion needed. The old cached results just become orphaned and eventually get cleaned up.

NOTE on league night pages: Per locked decisions, league night pages are NOT gated. They use `generateStaticParams` to build pages for every week with data. Do NOT modify any queries in the week page route.
