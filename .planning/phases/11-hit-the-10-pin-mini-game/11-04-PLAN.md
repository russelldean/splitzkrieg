    9. `pin-machine` (tier 2): The pin setter machine descends and scoops up the pin before the ball arrives. Caption: "Maintenance window. Try again in 5-7 business days."
    10. `invading-ball` (tier 4): Another bowling ball rolls in from the next lane and knocks your ball into the gutter. Caption: "Lane 7 sends their regards."

    Each cheat's `execute` function is an async function that:
    1. Receives the game engine and renderer
    2. Manipulates physics (for physics cheats) or triggers visual animations (for character/bowling cheats)
    3. Returns a Promise that resolves when the animation completes (using setTimeout-based timing)
    4. Duration is set per cheat (1000-3000ms)

    For the `execute` implementations:
    - Physics cheats: Modify ball velocity mid-flight (`Body.setVelocity`) or modify world properties
    - Character cheats: Set a cheat animation state that the renderer reads via `drawCheatEffect(ctx, cheatId, progress)`. Progress goes from 0 to 1 over the duration. The renderer draws the character based on progress.
    - Bowling cheats: Mix of physics manipulation and visual effects

    The execute functions use a simple animation loop:
    ```typescript
    execute: async (engine, renderer) => {
      return new Promise<void>(resolve => {
        const start = performance.now();
        const animate = () => {
          const elapsed = performance.now() - start;
          const progress = Math.min(elapsed / duration, 1);
          // Store cheatId and progress for renderer to read
          if (progress < 1) requestAnimationFrame(animate);
          else resolve();
        };
        requestAnimationFrame(animate);
      });
    }
    ```
