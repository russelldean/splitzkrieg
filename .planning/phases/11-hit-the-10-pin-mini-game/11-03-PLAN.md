    1. **Pointer event handlers on canvas element:**
       - `onPointerDown`: If state.phase === 'idle', call `slingshotInput.onPointerDown(canvasPos)` and transition state to 'aiming'. Call `event.preventDefault()`.
       - `onPointerMove`: If state.phase === 'aiming', call `slingshotInput.onPointerMove(canvasPos)`. Draw aim arrow in render loop using `slingshotInput.getAimVector()` per D-02.
       - `onPointerUp`: If state.phase === 'aiming', get velocity from `slingshotInput.onPointerUp()`. If velocity is non-null, transition to 'rolling', call `engine.launchBall(velocity)` per D-06 (instant release). Call `event.preventDefault()`.

    2. **Convert pointer coordinates to canvas coordinates** accounting for canvas offset and HiDPI scaling:
       ```typescript
       function getCanvasPos(e: PointerEvent, canvas: HTMLCanvasElement): Vec2 {
         const rect = canvas.getBoundingClientRect();
         return { x: e.clientX - rect.left, y: e.clientY - rect.top };
       }
       ```

    3. **During 'rolling' phase in game loop:**
       - Call `updateCamera(camera, engine.getBallPosition().y, canvasHeight)` per D-07
       - Check `engine.isPinHit()` -- if true and `shouldWin(state.isAdmin)`, transition to 'win'
       - Check `engine.isBallOutOfBounds()` -- if true, select and trigger a cheat (for now, just transition to 'result' since cheats come in Plan 04). Increment attempt count.

    4. **During 'result' phase:**
       - After 1 second delay, check if attempts remain. If yes, `engine.resetBall()`, reset camera, transition to 'idle'. If no, transition to 'scorecard'.

    5. **Render the aim arrow** during 'aiming' phase:
       - Get `slingshotInput.getAimVector()`
       - If non-null, call `renderer.drawAimArrow(ctx, ballPosition, aimVector)`
       - Also render a visual pull-back indicator: a line from ball to current pointer position (rubberband visual)

    8. **Lying aim predictor (D-35):** While in 'aiming' phase, display text feedback about the player's aim quality that is ALWAYS wrong:
       - Calculate the actual aim quality (how close the trajectory would come to the pin)
       - If aim is good (would hit or nearly hit): show discouraging text like "Terrible aim!", "Not even close!", "Are you even trying?"
       - If aim is bad (way off): show encouraging text like "Definitely going to hit it this time!", "Perfect angle!", "This is the one!"
       - Text appears near the top of the canvas, updates in real-time as the player adjusts aim
       - Use the renderer interface: add `drawPredictorText(ctx: CanvasRenderingContext2D, text: string)` method to GameRenderer
       - Create a `getAimFeedback(aimVector: Vec2, ballPos: Vec2, pinPos: Vec2): string` pure function in a new file `src/components/game/AimPredictor.ts`
       - The function calculates how close the aim line would pass to the pin, then returns the OPPOSITE feedback
       - Keep a pool of 4-5 encouraging and 4-5 discouraging phrases, pick randomly each time aim changes significantly

    6. **Update GameEngine.ts** to add friction/damping so ball naturally slows down. The ball should roll with convincing weight per D-05. Add `setFriction` on the ball body's air friction to ~0.001 so it decelerates slowly. Also add:
       - `makePinDynamic()` method: switches pin from static to dynamic right before ball reaches pin area (so pin can be knocked)
       - Call `makePinDynamic()` when ball Y position is within 100px of pin Y
       - `onCollision` event via `Matter.Events.on(engine, 'collisionStart', ...)` to detect ball-pin contact

    7. **Handle D-04 (power from pull distance):** This is achievable without excessive complexity since `calculateLaunch` already scales velocity by pull distance. Keep it.
